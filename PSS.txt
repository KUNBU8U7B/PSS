==================================================
PSS LANGUAGE SPECIFICATION
PART 1 – CORE FOUNDATION
(Simple Programming Script)
==================================================

Dokumen ini menjelaskan dasar bahasa PSS dari nol
hingga dapat diparse, dijalankan, dan diuji.
Tidak mencakup function lanjutan dan OOP.
Semua aturan di PART 1 bersifat WAJIB.


==================================================
1. FILOSOFI BAHASA
==================================================

PSS (Simple Programming Script) adalah bahasa
pemrograman sederhana berbasis indentasi.

Tujuan utama:
- Mudah dipelajari
- Sintaks bersih
- Mudah ditranspile ke C
- Mudah dianalisis oleh AI

PSS dirancang agar:
- Tidak memakai tanda {} untuk block
- Menggunakan indent + ':' seperti Python
- Satu statement per baris


==================================================
2. STRUKTUR PROGRAM
==================================================

Setiap program PSS HARUS:

1. Diawali dengan keyword `program`
2. Diakhiri dengan keyword `end`

Bentuk umum:

program <nama_program>
    <isi_program>
end

Contoh VALID:
program demo
    print "halo"
end

Contoh TIDAK VALID:
print "halo"
(end tidak ada)


==================================================
3. BARIS & KOMENTAR
==================================================

- Satu baris = satu statement
- Baris kosong diabaikan
- Komentar diawali dengan '#'

Contoh:
# ini komentar
number x = 10   # komentar di akhir baris


==================================================
4. WHITESPACE & INDENTASI
==================================================

PSS adalah INDENT-BASED LANGUAGE.

Aturan indent:
- Indent menggunakan SPASI
- 1 level indent = 4 spasi
- Tab TIDAK disarankan

Indent menentukan:
- Awal block
- Akhir block

Kesalahan indent = SYNTAX ERROR


==================================================
5. KEYWORD RESMI
==================================================

Keyword yang dilindungi:

program
end
number
decimal
string
bool
byte
print
if
else
repeat
times
true
false

Keyword TIDAK boleh dipakai sebagai nama variabel.


==================================================
6. IDENTIFIER
==================================================

Aturan identifier:
- Huruf (a-z, A-Z)
- Angka (0-9)
- Underscore (_)
- Tidak boleh diawali angka

Contoh VALID:
x
nilai
total_harga

Contoh TIDAK VALID:
1x
nilai-total


==================================================
7. TIPE DATA DASAR
==================================================

PSS memiliki tipe data berikut:

number   -> bilangan bulat (int)
decimal  -> bilangan pecahan (double)
string   -> teks
bool     -> true / false
byte     -> bilangan kecil (0-255)

Contoh:
number a = 10
decimal pi = 3.14
string msg = "halo"
bool ok = true
byte b = 255


==================================================
8. DEKLARASI VARIABEL
==================================================

Bentuk:
<type> <nama> = <expression>

Contoh:
number x = 5
decimal luas = 3.14 * 10 * 10

Aturan:
- Variabel HARUS dideklarasikan sebelum dipakai
- Tidak boleh redeclare dalam scope yang sama


==================================================
9. ASSIGNMENT
==================================================

Bentuk:
<nama> = <expression>

Contoh:
x = x + 1

Type mismatch akan menghasilkan error.


==================================================
10. OPERATOR
==================================================

Aritmatika:
+  -  *  /  %

Perbandingan:
==  !=  <  >  <=  >=

Logika:
and  or  not


==================================================
11. EXPRESSION
==================================================

Expression adalah kombinasi:
- literal
- variabel
- operator

Contoh:
a + b * 2
x % 2 == 0

Precedence (tinggi ke rendah):
1. ()
2. * / %
3. + -
4. perbandingan
5. logika


==================================================
12. PRINT / OUTPUT
==================================================

Sintaks:
print <arg1>, <arg2>, ...

Contoh:
print "nilai =", x
print "luas =", luas

Print otomatis:
- number  -> %d
- decimal -> %f
- string  -> %s
- bool    -> true/false


==================================================
13. IF STATEMENT
==================================================

Sintaks:
if <kondisi>:
    <block>
else:
    <block>

Contoh:
if x > 10:
    print "besar"
else:
    print "kecil"

Else bersifat opsional.


==================================================
14. KONDISI
==================================================

Kondisi HARUS menghasilkan bool.

Contoh VALID:
if x == 10:
if x % 2 == 0:

Contoh TIDAK VALID:
if x + 1:


==================================================
15. LOOP DASAR
==================================================

Repeat loop:

repeat <number> times:
    <block>

Contoh:
number i = 0
repeat 5 times:
    print i
    i = i + 1

Nilai repeat HARUS number.


==================================================
16. SCOPE DASAR
==================================================

- Semua variabel di PART 1 bersifat GLOBAL
- Block if/loop tidak membuat scope baru
- Shadowing belum diperbolehkan


==================================================
17. ERROR MODEL DASAR
==================================================

Jenis error:

1. Syntax Error
   - indent salah
   - keyword salah
   - ':' hilang

2. Type Error
   - number + string
   - assign tipe berbeda

3. Runtime Error
   - pembagian dengan 0


==================================================
18. CONTOH PROGRAM VALID
==================================================

program demo

decimal pi = 3.14
number r = 5
decimal luas = pi * r * r

number i = 0
repeat 5 times:
    if i % 2 == 0:
        print "i =", i
    else:
        print "ganjil"
    i = i + 1

print "luas =", luas

end


==================================================
19. CONTOH PROGRAM TIDAK VALID
==================================================

program demo
number x = "halo"
end

Error:
Type mismatch (number != string)

==================================================
20. FUNCTION (PROSEDURAL)
==================================================

Function adalah blok kode yang dapat dipanggil ulang.

Sintaks umum:

function <nama>(<param1>, <param2>, ...) -> <return_type>:
    <block>

Contoh:
function tambah(a, b) -> number:
    return a + b

Aturan:
- Nama function mengikuti aturan identifier
- Parameter bersifat lokal
- Function boleh mengembalikan nilai


==================================================
21. RETURN STATEMENT
==================================================

Sintaks:
return <expression>

Contoh:
return x * 2

Aturan:
- Tipe return HARUS sesuai deklarasi
- Function tanpa return dianggap error
  kecuali return_type = void


==================================================
22. FUNCTION VOID
==================================================

Function tanpa nilai balik menggunakan tipe `void`.

Contoh:
function tampil(msg) -> void:
    print msg

Pemanggilan:
tampil("halo")


==================================================
23. PEMANGGILAN FUNCTION
==================================================

Bentuk:
<nama_function>(<arg1>, <arg2>, ...)

Contoh:
number hasil = tambah(2, 3)

Aturan:
- Jumlah argumen HARUS sama
- Tipe argumen HARUS cocok


==================================================
24. RECURSIVE FUNCTION
==================================================

Function boleh memanggil dirinya sendiri.

Contoh:
function faktorial(n) -> number:
    if n <= 1:
        return 1
    else:
        return n * faktorial(n - 1)


==================================================
25. SCOPE LOKAL
==================================================

Scope baru dibuat oleh:
- function
- class method

Aturan:
- Variabel di dalam function bersifat lokal
- Variabel global bisa dibaca, tapi tidak ditimpa
  tanpa deklarasi ulang


==================================================
26. SHADOWING
==================================================

Variabel lokal boleh menimpa nama global.

Contoh:
number x = 10

function test() -> void:
    number x = 5
    print x   # 5

print x       # 10


==================================================
27. MEMORY MODEL (KONSEPTUAL)
==================================================

Model memori PSS:

- Global variable  -> static memory
- Function local   -> stack
- Object / array   -> heap

Mapping ke C:
- number  -> int
- decimal -> double
- string  -> char*
- object  -> struct pointer


==================================================
28. ARRAY
==================================================

Deklarasi array:

number[] data = [1, 2, 3, 4]

Akses:
print data[0]

Assignment:
data[1] = 10

Aturan:
- Index dimulai dari 0
- Out-of-bounds = runtime error


==================================================
29. ARRAY LOOP
==================================================

Array bisa dipakai di loop.

Contoh:
repeat 4 times:
    print data[i]


==================================================
30. CLASS (OOP DASAR)
==================================================

Class adalah blueprint object.

Sintaks:
class <NamaClass>:
    <block>

Contoh:
class Player:
    number hp = 100


==================================================
31. OBJECT
==================================================

Membuat object:

Player p
p = new Player()

Akses properti:
print p.hp


==================================================
32. METHOD
==================================================

Method adalah function di dalam class.

Contoh:
class Player:
    number hp = 100

    function damage(x) -> void:
        hp = hp - x

Pemanggilan:
p.damage(10)


==================================================
33. CONSTRUCTOR
==================================================

Constructor bernama `init`.

Contoh:
class Player:
    number hp

    function init(start_hp) -> void:
        hp = start_hp

Pemanggilan:
Player p = new Player(100)


==================================================
34. ENCAPSULATION
==================================================

Modifier akses:

public   -> default
private  -> hanya di dalam class

Contoh:
class Bank:
    private number saldo

    function setor(x) -> void:
        saldo = saldo + x


==================================================
35. INHERITANCE
==================================================

Class dapat mewarisi class lain.

Sintaks:
class Child extends Parent:
    <block>

Contoh:
class Enemy extends Player:
    number damage = 10


==================================================
36. METHOD OVERRIDE
==================================================

Method child boleh menimpa method parent.

Contoh:
class Enemy extends Player:
    function damage(x) -> void:
        hp = hp - (x * 2)


==================================================
37. POLYMORPHISM (KONSEP)
==================================================

Object child dapat diperlakukan sebagai parent.

Contoh:
Player p
p = new Enemy()
p.damage(5)


==================================================
38. MODULE / IMPORT
==================================================

Import file PSS lain.

Sintaks:
import "math.pss"

Aturan:
- File di-parse satu kali
- Namespace mengikuti nama file

========================================================

5.0 TUJUAN DESAIN LIBRARY PSS
--------------------------------------------------------
PSS (Programming Simple Script) dirancang sebagai bahasa
pemrograman yang ringan, mudah dipelajari, dan dapat
berjalan di perangkat dengan spesifikasi rendah seperti
HP dan laptop lama.

Fokus utama desain:
- Core kecil
- Library modular
- Tidak memaksa fitur berat
- Mudah dipahami oleh pemula

Prinsip utama:
"CORE KECIL, FITUR BESAR LEWAT EKSTENSI"


5.1 KEBIJAKAN UKURAN (SIZE POLICY)
--------------------------------------------------------

5.1.1 Target Ukuran
- Compiler + runtime inti PSS:
  < 5 MB (target desain)
- Tanpa model AI bawaan
- Tanpa engine besar
- Tanpa dependency berat

5.1.2 Yang TIDAK Termasuk Core
Fitur berikut TIDAK BOLEH berada di core:

- Model AI / Machine Learning
- Large Language Model (LLM)
- Game engine besar
- Web framework kompleks
- Library scientific berat

Fitur di atas hanya boleh hadir sebagai:
- Plugin
- Library eksternal
- Download opsional


5.2 KONSEP LIBRARY DI PSS
--------------------------------------------------------
Library adalah kumpulan fungsi siap pakai agar programmer
tidak perlu menulis semuanya dari nol.

Library di PSS harus:
- Mudah dimengerti
- Nama jelas
- Minim simbol rumit
- Aman untuk pemula

Contoh penggunaan:
use math
use io


5.3 SINTAKS PENGGUNAAN LIBRARY
--------------------------------------------------------

5.3.1 Import Library
use nama_library

Contoh:
use math

5.3.2 Import Dengan Alias
use nama_library as alias

Contoh:
use math as m

Pemanggilan:
m.sqrt(25)

--------------------------------
7. SISTEM ERROR & WARNING
--------------------------------

7.1 Jenis Pesan
---------------
PSS hanya mengenal 3 jenis pesan:

1. ERROR   → program berhenti
2. WARNING → program jalan, tapi ada resiko
3. INFO    → informasi saja

Format umum:
[TYPE][FILE:LINE] MESSAGE

Contoh:
[ERROR][main.pss:12] variable 'x' not defined
[WARNING][loop.pss:8] implicit type conversion
[INFO][sys] transpiling started

--------------------------------
7.2 ERROR WAJIB (FATAL ERROR)
--------------------------------

Error yang WAJIB menghentikan proses:

- Indent salah
- Tipe data tidak cocok
- Variabel belum dideklarasikan
- Operator tidak valid
- Kurung tidak seimbang
- Keyword salah tulis

Contoh:
decimal x = "halo"
→ ERROR: type mismatch

--------------------------------
7.3 WARNING (NON-FATAL)
--------------------------------

Warning tidak menghentikan eksekusi:

- Konversi number → decimal
- Variabel tidak dipakai
- Loop tidak digunakan

Tujuan warning:
- Edukasi
- Bukan menghakimi user

--------------------------------
8. DEBUGGING MODE
--------------------------------

PSS menyediakan mode debug ringan:

pss run file.pss --debug

Output debug:
- Baris yang dieksekusi
- Nilai variabel penting
- Hasil evaluasi kondisi

Contoh output:
[DEBUG] i = 3
[DEBUG] condition (i % 2 == 0) → false

--------------------------------
9. TOOL BAWAAN PSS (RINGAN)
--------------------------------

Semua tool bersifat OPSIONAL dan kecil ukurannya.

9.1 pss check
-------------
Memeriksa syntax tanpa menjalankan program

pss check main.pss

9.2 pss format
--------------
Meratakan indent otomatis

pss format main.pss

9.3 pss transpile
-----------------
Mengubah PSS ke target bahasa (C / Python)

pss transpile main.pss --target=c

--------------------------------
10. FORMAT OUTPUT UNTUK AI HELPER
--------------------------------

PSS mendukung output yang RAMAH AI,
tanpa perlu AI berat di dalam bahasa.

Format standar error untuk AI:

{
  "type": "error",
  "line": 12,
  "code": "E_TYPE_MISMATCH",
  "message": "decimal cannot receive string"
}

AI eksternal cukup membaca JSON ini.
